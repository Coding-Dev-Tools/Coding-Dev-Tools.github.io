Convert between 11 schema formats without copying by hand

Migrating an ORM is mostly copying fields by hand. I went through this moving from Prisma to Drizzle. A week of diffing files, fixing type mismatches, and wondering if I missed a relation.

SchemaForge handles the conversion. Point it at a schema in any supported format and it converts to any of the other 10. That is 11 formats and 110 direction pairs with zero-loss roundtripping.

Install

pip install git+https://github.com/Coding-Dev-Tools/schemaforge.git
schemaforge convert --from prisma --to drizzle --input schema.prisma

What it does

  • converts between 11 formats (SQL DDL, Prisma, Drizzle, TypeORM, Django, SQLAlchemy, Alembic, JSON Schema, GraphQL SDL, EF Core C#, Scala)
  • zero-loss roundtripping — the same schema converts back and forth without drift
  • diff mode for comparing two schemas in the same format
  • VS Code extension for live preview and quick conversion
  • MCP server for AI agents that need schema work
  • 310 passing tests
  • Python 3.10+, MIT license

Why it works

Every format converts to and from a shared internal representation. That is what makes roundtripping safe and why adding a new format does not require pairwise converters. You add one parser and one generator.
