Monitor service health with a Python CLI — no dashboard required

profitmax is a local Python agent that checks services and suggests fixes with an LLM. No hosted dashboard, no recurring bill.

Install

git clone https://github.com/Coding-Dev-Tools/profitmax-pro.git
cd profitmax-pro
pip install .
profitmax --task status

What it checks

It ships with health-check tools for OpenClaw, Paperclip, KiloClaw, and an aggregated service-health view. The LLM layer uses a fallback chain: DeepSeek, then Gemini Flash, then Mistral. Checks run in the order you register them, not in parallel.

Add your own checks

The registry is decorator-based. Add a custom check with a function and a decorator instead of editing a central config file.

from profitmax import registry

@registry.health_check
def payment_gateway():
    return {"ok": True, "latency_ms": 180}

When it is useful

  • local stacks where a hosted monitor feels heavy
  • CI or staging runs that need a quick health snapshot
  • teams that already have dashboards but want a suggested-fix layer

Honest limits

This is not a monitoring dashboard and not an alerting service. It runs when you run it. The LLM suggestions are only as good as the checks and the model you have configured locally.
